# PRD: 캐봄 (Kebom)

## 목표
취준생이 이력서와 채용공고의 간극을 파악하지 못해 서류 탈락하는 문제를 해결한다.
AI가 이력서를 "제대로 읽었는지" 사용자가 직접 확인할 수 있는 신뢰 기반 위에서, 단순 오답 노트를 넘어선 실행 가능한(Actionable) 합격 전략을 제시한다.

## 사용자
- 서류 탈락의 이유를 몰라 막막한 취준생
- 지원 전 최종 점검을 원하는 지원자

## 핵심 기능

### Step 1: Read — AI 인식표
1. 사용자가 채용공고(텍스트 입력 또는 이미지 업로드)와 이력서(PDF)를 업로드한다.
2. AI가 이력서에서 추출한 핵심 역량/경력/프로젝트 요약을 먼저 보여준다.
3. 사용자가 AI 파싱 결과를 확인하고 틀린 부분을 수정할 수 있다.

### Step 2: Analyze — 매칭 히트맵
1. 채용공고의 요구사항과 이력서를 비교하여 매칭(보유)/미흡(보완 필요)/공백(없음) 3단계로 시각화한다.
2. 전체 매칭 점수(0~100)를 표시한다.
3. 각 스킬에 대해 이력서에서 발견된 근거 또는 구체적인 갭을 함께 표시한다.

### Step 3: Action — 전략 제안
1. **Danger Zone**: 면접관 관점에서 태클 걸릴 만한 약점 질문 리스트 (최대 5개) + 대응 방향 제시.
2. **Magic Fix**: 공고 맞춤형 키워드 삽입 및 문장 수정 가이드 (최대 5개).

## MVP 제외 사항
- 사용자 인증 / 로그인
- 분석 결과 저장 / 히스토리
- 이력서 PDF 직접 편집 및 재생성
- 여러 공고 동시 비교
- 모바일 앱

## 제약 및 지원 범위

### 파일 형식
- **이력서**: PDF 전용 (최대 5MB). 텍스트 레이어 필수 — 스캔 이미지 PDF 미지원.
- **JD 이미지**: PNG, JPG(JPEG) 전용 (최대 5MB). WebP, GIF, BMP, SVG 미지원.

### 최소 지원 브라우저
640px 이상 뷰포트를 갖춘 모던 브라우저:
- Chrome 120+, Firefox 121+, Safari 17+, Edge 120+
- Internet Explorer 미지원. 모바일 앱 MVP 제외.
- `navigator.clipboard` 미지원 환경: Magic Fix 복사 버튼 미노출. 상세: docs/UI_GUIDE.md 참조.

### 네트워크 재시도 정책
자동 재시도 없음. 타임아웃·에러 시 사용자가 수동으로 재시도 버튼 클릭.
백오프·자동재시도·오프라인 지원 MVP 제외.

### Danger Zone advice 포맷
각 면접 위험 질문(`DangerQuestion`)의 `advice` 필드: 한국어 2–4문장.
"~하세요" 명령형 또는 "~를 강조하세요" 제안형 중 AI 판단에 따라 선택.

## 에러 시나리오

| 상황 | 사용자에게 표시할 메시지 | 처리 방식 |
|------|--------------------------|-----------|
| PDF 파싱 실패 (스캔 이미지 PDF 등) | "이력서를 읽을 수 없습니다. 텍스트 기반 PDF를 사용해주세요." | 파일 재업로드 유도 |
| 파일 크기 초과 (5MB) | "파일 크기는 5MB 이하여야 합니다." | 업로드 차단, 안내 메시지 |
| 지원하지 않는 파일 형식 | "이력서는 PDF 파일만 지원합니다." | 업로드 차단 |
| AI API 일반 오류 | "분석 중 오류가 발생했습니다. 잠시 후 다시 시도해주세요." | 재시도 버튼 제공 |
| AI 응답 타임아웃 (>30초) | "분석이 너무 오래 걸리고 있어요. 잠시 후 다시 시도해주세요." | 재시도 버튼 제공. 클라이언트 AbortController 30초. 타임아웃 UX 상세: docs/UI_GUIDE.md 참조. |
| 채용공고 텍스트 미입력 | "채용공고를 입력하거나 이미지를 업로드해주세요." | 제출 차단 |
| 채용공고에 식별 가능한 스킬 없음 | "채용공고에서 요구 스킬을 찾을 수 없습니다. 공고 내용을 확인해주세요." | 파싱은 완료. StepRead 인라인 경고 표시. 분석 진행은 허용. |
| 이력서에 스킬/경력 없음 (연락처만 존재) | "이력서에서 경력 또는 스킬 정보를 찾을 수 없습니다. 직접 추가해주세요." | StepRead 진입 후 인라인 경고. 사용자가 직접 추가 후 진행 가능. |
| 이미지 JD 파일 크기 초과 (5MB+) | "채용공고 이미지는 5MB 이하여야 합니다." | 업로드 차단, 안내 메시지 |
| 지원하지 않는 이미지 형식 (GIF, WebP, BMP 등) | "PNG 또는 JPG 이미지만 업로드 가능합니다." | 업로드 차단 |
| 이미지에서 텍스트 인식 불가 (그림만 있는 이미지) | "이미지에서 채용공고 텍스트를 인식할 수 없습니다. 텍스트를 직접 입력해주세요." | 텍스트 입력 유도 (내부 500) |
| 비밀번호로 보호된 PDF | "비밀번호로 보호된 PDF는 읽을 수 없습니다. 보호를 해제 후 업로드해주세요." | 파일 재업로드 유도 |
| 손상된(corrupted) PDF 파일 | "PDF 파일이 손상되어 읽을 수 없습니다. 다른 파일을 사용해주세요." | 파일 재업로드 유도 |
| AI가 유효하지 않은 JSON 반환 | "분석 중 오류가 발생했습니다. 잠시 후 다시 시도해주세요." | 재시도 버튼 제공 (내부 500 처리) |
| 네트워크 연결 끊김 (fetch 자체 실패) | "네트워크 연결을 확인해주세요." | 재시도 버튼 제공 |
| JD 텍스트와 이미지 동시 제출 | 별도 사용자 메시지 없음 | 텍스트 우선 처리, 이미지 무시 |
| StepRead 이름/이메일 필수 필드 비어있음 | "이름과 이메일은 필수 항목입니다." | 분석 버튼 비활성화, 필드 인라인 에러 표시 |
| 드롭존에 여러 파일 드롭 | "파일은 한 번에 하나만 선택할 수 있습니다." | 첫 번째 파일만 수락 |

## 엣지 케이스

| 상황 | 처리 정책 |
|------|----------|
| 브라우저 뒤로 가기 버튼 | 페이지 상태 완전 초기화 (세션 스토리지 미사용 — Step 1부터 재시작) |
| Step 3 완료 후 새 분석 시작 | "새 분석 시작" 버튼 → `dispatch(RESET)` → Step 1(upload)로 복귀 |
| 드롭존에 PDF 아닌 파일 드롭 | 클라이언트에서 즉시 차단. "이력서는 PDF 파일만 지원합니다." 인라인 표시 |
| JD 텍스트와 이미지 동시 제출 | 텍스트 우선 처리. 이미지 무시. 서버에서 `jobDescription` 존재 시 `jobImage` 필드 무시. |
| 스킬 배열 중복 항목 입력 | 클라이언트에서 중복 허용 (사용자가 직접 삭제 가능). 서버 측 중복 제거 없음. |
| Magic Fix 클립보드 미지원 브라우저 | `'clipboard' in navigator` 체크 → 미지원 시 복사 버튼 미노출. UI_GUIDE.md 참조. |

## 로딩 UX

| 단계 | 로딩 메시지 | 예상 소요 시간 | 타임아웃 |
|------|-------------|----------------|---------|
| 파싱 중 (Step 1→2) | "이력서와 공고를 읽고 있어요..." | < 10초 (Sonnet) | 10초 |
| 분석 중 (Step 2→3) | "강점과 약점을 분석하고 있어요..." | < 30초 (Opus) | 30초 |

로딩 중에는 버튼 비활성화, 스켈레톤 UI 표시. 진행 중임을 명확히 전달.
15초 경과 시 보조 텍스트 "(시간이 더 걸릴 수 있어요)" 추가 표시 (분석 단계만).
타임아웃 UI 상세(AbortController, 보조 메시지 타이밍, 스켈레톤 구조): docs/UI_GUIDE.md 참조.
